# 公众号文章生成器 (gen-article)

## 描述
通用公众号文章生成技能。用户输入类目、字数、风格，自动完成从素材抓取到HTML产出的全流程。

## 触发方式
用户输入 `/gen-article` 或说"写公众号文章"

## 执行流程

### 第0步：收集参数（必须先问用户）

用 AskUserQuestion 工具询问以下信息，如果用户已经提供则跳过：

1. **文章类目**（必填）：如 情感、科技、职场、育儿、美食、故事、生活 等
2. **字数要求**（必填）：如 300字、500字、800字
3. **文章风格**（必填）：
   - 倾诉型：像跟朋友聊天，口语化，第一人称
   - 故事型：完整故事线，有起承转合
   - 干货型：条理清晰，有知识点
   - 观点型：有立场，有论证
4. **目标平台**（可选，默认微信公众号）
5. **特殊要求**（可选）：如"要有争议性""要正能量"等

### 第1步：素材抓取

根据用户输入的类目，决定搜索关键词和抓取源。

**小红书抓取**（优先）：
```bash
cd ARTICLE_GEN_DIR
python scripts/fetch_xhs.py --keywords "关键词1,关键词2"
```

**知乎抓取**（补充深度内容）：
```bash
python scripts/fetch_zhihu.py --max 10
```

**关键词生成规则**（根据类目自动扩展）：
| 类目 | 搜索关键词示例 |
|------|---------------|
| 情感 | 暗恋 分手 心事 树洞 婚姻 出轨 |
| 科技 | AI 编程 程序员 互联网 产品 |
| 职场 | 加薪 跳槽 领导 同事 内卷 |
| 育儿 | 宝妈 带娃 教育 学区房 亲子 |
| 美食 | 做饭 家常菜 烘焙 外卖 厨房 |
| 故事 | 真实经历 树洞 匿名 投稿 |
| 生活 | 独居 租房 搬家 记账 省钱 |

### 第2步：筛选加工

- 过滤与类目相关的关键词
- 清理文本（去话题标签、表情、广告、引流话术）
- 去重
- 保留有故事性/信息量的内容

### 第3步：改写为文章

**根据用户选择的风格改写：**

**倾诉型**：
- 第一人称"我"，口语化
- 有具体场景、对话、细节
- 短段落，手机阅读友好

**故事型**：
- 完整故事线：起因→经过→高潮→结局
- 有人物、事件、冲突
- 有转折或反转

**干货型**：
- 条理清晰，分点论述
- 有数据、案例支撑
- 实用性强

**观点型**：
- 有明确立场，有论据
- 引发思考或讨论
- 结尾抛出问题

**通用要求：**
- 字数严格按用户要求（±10%）
- 去AI味，说人话
- 结尾留互动钩子

### 第4步：合规审查

**通用规则：**
- 无政治敏感内容
- 无色情低俗
- 无虚假信息/谣言
- 无诱导分享/关注
- 无夸大标题党

**类目专项规则：**
| 类目 | 特别注意 |
|------|----------|
| 情感 | 不美化出轨，不煽动性别对立 |
| 科技 | 不传播不实技术信息 |
| 职场 | 不煽动劳资对立 |
| 育儿 | 不传播伪科学 |
| 美食 | 不传播食品安全谣言 |
| 故事 | 标注"投稿/虚构" |

### 第5步：配爆款标题

**通用标题公式：**
- [具体数字/细节] + [情感/悬念]
- [场景] + [反转]
- [身份] + [痛点]
- [问题] + [好奇心]

**要求：** 15-20字，有具体场景，不夸大，与内容一致

### 第6步：HTML排版

**排版规范：**
- 字体：PingFang SC / Microsoft YaHei
- 正文15px，标题20px居中加粗
- 行高2.0，字色#3f3f3f
- 主题色#d9a4a0（可自定义）
- 金句高亮：左侧竖线+斜体
- 页尾签名：账号名+slogan

**输出位置：** 用户指定目录（默认 ./output/）
- 每篇单独 .html + .txt
- 文件名 = 文章标题
- 配图放 images/ 子文件夹

### 第7步：配图

**来源：** Unsplash（免费商用）
**选图原则：** 与文章核心场景强关联的写实画面

## 快捷指令
- `/gen-article 情感 300字 倾诉型`
- `/gen-article 科技 500字 干货型`
- `/gen-article 故事 800字 故事型`
